# 栈

# 基本概念

操作受限的线性表

概念：栈顶、栈底、空栈

操作特性：后进先出（Last In First Out,LIFO）

卡特兰数：，n 个不同元素进栈，不同出栈排列个数

# 顺序存储结构

栈顶指针 s.top ，初始值 -1（0 则是规定指向栈顶元素的下一个存储单元）

进栈：不满，栈顶指针 + 1，再送值到栈顶元素

出栈：非空，取元素值，栈底指针 - 1

判空： s.top == -1

判满： s.top == MaxSize-1 （数组）

栈长： s.top + 1

# 共享栈

两个顺序栈共享一维数组，两端栈底，栈顶双向奔赴。

判满：右栈顶与左栈顶差值为一

判空：左栈顶 - 1，右栈顶 MaxSize

# 顺序存储结构

# 顺序存储

头 front 指针指队头元素，尾指针 rear 指队尾元素的下一个位置

判空：队头队尾相等为 0

进队：不满，先送值至队尾，队尾指针 + 1

出队：不空，先取队头元素，队头指针 + 1

注意 Q.rear == MaxSize 无法判满，可能出现上溢出，实际上还有内存空间

# 循环队列

逻辑上视顺序队列为环状，出队入队均为顺时针方向进 1

# 队空 / 满

  1. 牺牲一个单元区分，队头指针在队尾指针下一位置为队满
    • 队满 (Q.rear+1)%MaxSize == Q.front
    • 队空 Q.front == Q.rear
    • 队中元素个数 (Q.rear-Q.front+MaxSize)%MaxSize
  2. 类型中增设表示元素个数的数据成员
    • 队空 Q.size==0 队满 Q.size==MaxSize
  3. 类型中增设 tag 数据成员区分， tag==0 时，因删除导致 Q.front == Q.rear 队空；等于 1 时，因插入导致，队满。
