概述

# 基本概念和术语

数据 信息的载体，描述客观事物属性的数、字符及所有能输入到计算机并被程序识别和处理的符号的集合。是程序加工的原料。

数据元素 数据的基本单位，由若干个数据项组成，数据项是构成其的不可分割的最小单位。

  • 如一条学生记录为一个数据元素，由学号、姓名、性别等数据项组成。

数据对象 具有相同性质的数据元素的集合。

数据类型 是一个值的集合和定义在该集合上的一组操作的总称。

  • 原子类型 其值不可再分的数据类型；
  • 结构类型 其值可在分解为若干成分的数据类型；
  • 抽象数据类型 抽象数据组织及与之相关的操作。

# 什么是数据结构

并无官方明确定义，可简述为相互之间存在一种或多种特定关系的数据元素的集合。数据元素在具体问题中并非孤立存在，其相互之间的关系称之为结构。数据结构包括三方面内容（即数据结构三要素）：

  • 逻辑结构；
  • 存储结构；
  • 数据的运算。

# 逻辑结构

元素之间的逻辑关系，从逻辑上描述，独立于计算机，与数据的存储无关。分为：

  • 线性结构（代表：线性表）；
  • 非线性结构（代表：集合、树、图）。

# 存储结构

指数据结构在计算机中的表示（又称映像），也称物理结构。包括数据元素的表示和关系的表示。依赖于计算机语言，主要的存储结构有：

  • 顺序存储 逻辑上相邻，物理上也相邻。可实现随机存储（根据一个元素所占空间计算出元素位置），每个元素占用最小的空间。只能使用相邻的一整块存储单元，故可能产生较多外部碎片；
  • 链式存储 逻辑上相邻，物理上不一定。由指针表示元素间逻辑关系，不会产生碎片现象，充分利用所有存储单元。每个元素存储指针占用额外存储空间，且只能实现顺序存取；
  • 索引存储 存储元素信息同时，建立附加索引表。表中每项称为索引项，其形式为 (关键字,地址) 。检索速度快，附加的索引表额外占用存储空间，增加、删除数据需要修改索引表，花费较多时间；
  • 散列存储 根据元素关键字甲酸元素存储地址，也称哈希存储。检索、增加、删除结点的操作都很快。若散列函数不好会导致元素存储单元冲突，解决冲突将会增加时间和空间开销。

# 数据的运算

包括运算的定义和实现。定义针对逻辑结构，指出运算的功能；实现针对存储结构，指出具体操作步骤。

# 什么是算法

简言之，算法就是解决问题的一种方法、思想。

算法有简单复杂之分，也有高效低能之分。衡量标准如下：

  • 时间复杂度；
  • 空间复杂度。

我们可以在查找、排序、最优决策等各领域看到算法大放光芒。

算法的设计取决于所选的逻辑结构，实现依赖于采用的存储结构。

线性结构 最简单的数据结构，可以看到它们是一对一。数组、链表及其衍生出来的栈、队列、哈希表。

# 算法的复杂度

衡量程序好坏的标准。简单来说，就是代码运行时间和程序的大小。
